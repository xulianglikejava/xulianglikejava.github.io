### 项目一：通威 ERP3.0 项目

**系统架构**：金蝶云苍穹+Spring Cloud Alibaba+Redis+PostgreSql+RabbitMQ+ZooKeeper+Dubbo
**开发时间**：2023.4 – 至今  
**项目描述**：  
通威 ERP 系统采用金蝶云苍穹系统搭建架构，集成了供应链、采购、财务、质量、制造、物流等多个核心模块。该系统运用先进的微服务架构，借助 Zookeeper 完成服务注册，通过 Dubbo 实现服务治理与负载均衡，确保系统具备高可用性、高性能和可扩展性。
**责任描述**：  
1. 负责模块：（1）深度负责供应链、TMS（运输管理系统）、SRM（供应商关系管理系统）、WMS（仓储管理系统）等模块的业务需求梳理、沟通对接与功能模块开发工作。与业务团队紧密协作，精准把握业务流程与需求痛点，将业务需求转化为可实现的技术方案，确保各模块功能贴合实际业务场景。
参与帆软开发工作，基于帆软工具进行数据报表与可视化界面的设计与开发。结合业务需求，运用帆软的报表设计器、仪表板等功能，构建直观、易懂的数据分析报表与可视化界面，为企业决策提供有力的数据支持。
2. 性能提升：（1）数据库调优工作，通过对数据库表结构、索引、查询语句等方面的优化，提升系统的查询性能与响应速度。运用数据库性能分析工具，深入分析数据库运行状态，找出性能瓶颈并制定针对性的优化方案（2）基于RabbitMQ，解决多个系统耦合严重问题。通过消息队列实现系统间的异步通信与解耦，提高系统的可扩展性与稳定性。
3. 数据治理：分布式事务，保证多个系统数据数据一致性。运用合适的分布式事务解决方案，保证在跨服务、跨数据库的业务操作中，数据的完整性与一致性。例如，在采购、库存等模块的业务流程中，确保数据的准确同步与更新，避免出现数据不一致的问题。

### 项目二：中科星图智慧教具项目

**系统架构**： SpringBoot +Vue +MySql + Netty + ffmpeg 
**开发时间**：2022.4 – 2023.4
**项目描述**：  
中科星图智慧教具聚焦于校园地理教育领域，通过创新性地运用传感器、摄像头等多元硬件设备，实现对地理实验的多维度支撑，为学生提供更加直观、全面的地理学习体验。在技术架构方面，服务端基于 Spring Boot 框架搭建，以保障系统的高效稳定运行；前端采用 Vue3 构建交互界面，为用户带来流畅的操作体验；借助 WebRTC 技术实现摄像头直播服务，满足远程教学与实验展示的需求；利用 Netty 框架实现设备间的稳定通信，确保数据传输的及时性与准确性。 

**责任描述**：  
1. 核心设计
- 主导技术方案选型工作，综合考量项目需求、性能要求、可扩展性等因素，审慎评估并选定适用于本项目的硬件设备，如高精度的地理传感器、高清摄像头等，确保硬件性能能够满足多维度地理实验数据采集与传输的需求。
- 针对项目所需的中间件进行选型与配置，如消息队列、缓存中间件等，以优化系统架构，提升系统的整体性能与稳定性。同时，精心设计数据库的库表结构，依据地理实验数据的特点与业务流程，合理规划数据存储方式，确保数据的高效存储与便捷查询。
- 深度参与业务流程设计，与教育领域专家、一线教师紧密沟通协作，充分理解地理教学的实际需求与实验流程，将其转化为合理、流畅的业务逻辑，确保系统功能能够精准贴合校园地理教育场景。
2. 项目管理：
- 全面负责项目管理工作，制定详细、合理的项目计划，明确各阶段的任务目标、时间节点与责任人。运用项目管理工具与方法，对项目进度进行实时跟踪与监控，及时发现并解决项目推进过程中出现的问题与风险，确保项目按计划顺利推进，按时交付高质量的产品。
3. 代码审查：
- 积极组织代码审查活动，定期召集开发团队成员对代码进行全面审查。在审查过程中，严格遵循代码规范与最佳实践，对代码的结构、逻辑、可读性、可维护性等方面进行细致评估，及时发现并指出代码中存在的潜在问题与缺陷。

### 项目三：中国电信安徽动力环境系统

**系统架构**：SpringBoot + JQuery + Vue + Kafka + MySQL + SVN + Redis + ElasticSearch + XXL-Job  
**开发时间**：2020.10 – 2022.4  
**项目描述**：  
中国电信安徽动力环境系统，承担全省空调、机房环境、油机等动力设备的管理、监控与告警任务。项目采用 Spring Boot+Vue+MySQL 搭建应用层，支撑用户管理设备；借助 XXL - Job、Redis、ElasticSearch，实现采集器对设备信号的监控与存储；运用 Kafka 高效推送告警信息 ，保障系统稳定运行。

**责任描述**：  
1. 性能提升, 核心查询服务：  
   - 使用内存数据库 Redis，存储热点数据；  
   - 基于减少长事务原则，按功能拆分原长 SQL 查询，按结果类别查询，拆分接口返回；  
   - 新建合适索引；  
   - 分析语句，把原先在语句中的计算函数抽出来，用 Java 代码实现。  
2. 数据管理服务：使用 ElasticSearch 对全省设备信号量、操作日志进行储存和管理，更方便对现场问题进行回滚定位；  
3. 页面重构：使用 Element-Ui 对原有页面进行优化；  
4. 负责项目上云工作；
